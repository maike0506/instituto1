3. Configuración a bajo nivel. Mediante comandos

Es posible hacer una configuración de la red con los comandos ifconfig, route e ip, es lo que se denomina configuración a bajo nivel, pero el inconveniente de este tipo configuraciones es que no se mantienen al reiniciar el sistema, a menos que se ejecuten desde un script en el arranque.
Los comandos ifconfig y route (paquete net-tools) están hoy en día en desuso y han sido sustituidos por el comando ip (paquete iproute).
Usaremos estos comandos para obtener información de la red y no para configurarla salvo en situaciones especiales como por ejemplo en el caso del mapeo lógico de un dispositivo de red.
Para ver la configuración de las interfaces de red activas ejecutamos:


# ifconfig

Se podrían ver todas las interfaces de red que tiene el sistema, tanto las activas (llevan la palabra UP) como las inactivas (no llevan la palabra UP), para lo cual, ejecutamos


# ifconfig -a

Podemos ver una estadística de la información enviada y recibida por las interfaces activas o todas:


# ifconfig -s

Podemos ver los datos de una interfaz concreta


# ifconfig eth0

Para mostrar la tabla arp usamos el comando arp:

# arp -n

La opción -n hace que se muestren todos los datos de forma numérica y no se intente hacer traducción de IP a nombre; esto hace, generalmente, que el comando funcione más rápido que obviando la opción.
La opción -a muestra la misma información con otro tipo de formato de salida. Para mostrar las tablas de enrutamiento usamos el comando route:

# route -n

También se puede utilizar la orden ip para obtener la información de la red. Así, para ver la lista de tarjetas que hay instaladas en el sistema, con su dirección MAC y si están activas o no, ejecutamos:

# ip link show

El parámetro show puede suprimirse, en la orden anterior y en las sucesivas, se asume por defecto.
Para ver las direcciones IP de las tarjetas:

# ip address show / es lo mismo que ip a

Configurafcion de las tarjetas a bajo nivel a traves ifconfig

En este anexo se va a resumir cómo se pueden utilizar los comandos ifconfig e ip para configurar las tarjetas de red.
Este método se utiliza cuando no es posible mediante otros modos.
La sintaxis resumida de ifconfig para asignar una IP a un tarjeta de red es la siguiente:

ifconfig intefaz dirección-ip netmask máscara [up]

Si se especifica up al final del comando, entonces la tarjeta se levanta a la vez que se le asigna la dirección IP. Por ejemplo:


# ifconfig eth0 192.168.0.2 netmask 255.255.255.0 up

Con los siguientes comandos se activa o desactiva la tarjeta de red:


ifconfig (interfaz) { up | down }

3.2. Configuración de las tarjetas a bajo nivel a través de ip

El comando ip realiza las mismas funciones que los comandos ifconfig, route y arp a la vez. Para poner una dirección IP, la sintaxis resumida sería


ip address add dirección-ip/prefijo dev interfaz

# ip address add 192.168.0.2/24 dev eth0

Si queremos cambiar de IP es necesario borrar previamente la dirección anterior para no asignar dos direcciones. La sintaxis para borrar una dirección IP es:


ip address del dirección-ip/prefijo dev interfaz # ip address del 192.168.0.2/24 dev eth0

Por último falta ver la sintaxis del comando ip para activar y desactivar la tarjeta de red:


ip link set interfaz { up | down }
# ip link set eth0 down
# ip link set eth0 up
